# Stratorama — HA Agent Add-on

A small Home Assistant add-on that links your local HA instance to your
[Stratorama](https://stratorama.app) account
through a secure outbound tunnel — no port forwarding, no token exchange,
no CORS to fight.

## How it works

The add-on opens a single outbound WebSocket to the tunnel server. After
a one-time pairing (you paste a short code generated by the app), the
tunnel can forward HA REST calls and stream HA `state_changed` events
through this connection.

```
[Stratorama app]  ──HTTPS──▶  [tunnel server]  ◀──WebSocket out──  [this add-on]
                                                                                  │
                                                                                  ▼ http://supervisor/core
                                                                          [Home Assistant]
```

## Setup

### 1. Add this repository to your Home Assistant

[![Open your Home Assistant instance and show the dialog to add a repository to the add-on store.](https://my.home-assistant.io/badges/supervisor_add_addon_repository.svg)](https://my.home-assistant.io/redirect/supervisor_add_addon_repository/?repository_url=https%3A%2F%2Fgithub.com%2Fstratorama%2Fstratorama-addon)

Click the button above and confirm the "Add Repository" dialog in your
Home Assistant — `my.home-assistant.io` auto-fills the URL.

**Manual fallback**: in Home Assistant, go to **Settings → Add-ons →
Add-on Store**, open the 3-dot menu (top right) → **Repositories** →
add this repository's URL:

```
https://github.com/stratorama/stratorama-addon
```

### 2. Install Stratorama - Agent

The add-on appears in the store under the "Stratorama Add-ons" section.
Click **Install**.

### 3. Pair with your account

In the Stratorama app, open the Settings menu → **Home Assistant**
→ **Générer un code**. You'll get an 8-character code valid 10 minutes.

In this add-on's **Configuration** tab:

- `pairing_code`: paste the code (one-shot — only used the first time).
- `tunnel_url`: keep the default unless you self-host the tunnel server.

Save and **Start** the add-on. Within seconds the app's status chip flips
to **✅ Connecté**.

After the first successful pairing, the add-on stores a long-lived
`agent_token` in `/data/agent-token.json`. You can clear the
`pairing_code` field afterwards — restarts will reconnect transparently.

### 4. Link entities to your floor plan

Open an element on the floor plan in edit mode → pen icon → pick a
matching HA entity in the dropdown (currently roller shutters / `cover.*`
in v1).

## Configuration reference

| Option         | Type   | Required | Description                                      |
|----------------|--------|----------|--------------------------------------------------|
| `pairing_code` | string | first run only | Generated by the app                       |
| `tunnel_url`   | url    | yes      | wss URL of the tunnel server                     |

## Permissions

The add-on declares `homeassistant_api: true`, which gives it:
- access to `http://supervisor/core/api/...` (REST)
- access to `ws://supervisor/core/api/websocket` (WS for state events)
- the `SUPERVISOR_TOKEN` env var, auto-injected by HA

It only makes **outbound** connections — it doesn't open any port on your
network.

## Logs & troubleshooting

Logs are visible in the add-on's **Log** tab. Common cases:

- `Code invalide / Code déjà utilisé / Code expiré` → generate a fresh code.
- `Token agent invalide` → the add-on's stored token was revoked (the user
  may have re-paired from another instance). Set `pairing_code` again with
  a fresh code — the add-on will rotate to a new token.
- `HA WS auth invalid` → the supervisor token isn't reaching the add-on.
  Confirm `homeassistant_api: true` is set in `config.yaml` (it is by
  default; relevant only if you forked).

## Repository layout

The add-on lives in `./stratorama_agent/` (a sub-folder is required by
the HA Supervisor — it scans subdirectories for `config.yaml`). A
`repository.yaml` at the root marks this repo as a HA add-on
repository.

## Building locally

For development:

```bash
cd stratorama_agent
docker build --build-arg BUILD_FROM=ghcr.io/home-assistant/amd64-base:3.20 -t stratorama-agent:dev .
```

The CI workflow at `.github/workflows/builder.yml` produces multi-arch
images (aarch64, amd64) on every push to `main` (armv7 dropped — too
slow under QEMU emulation on free GHA runners).
